# Website performance optimization implementation phase 1

Created: 2026-05-26

## Scope

Started implementation of `106 - Website performance optimization audit and plan - 2026-05-26.md`.

This phase targets the highest-impact work that can be completed without redesigning the site or changing the visible layout:

- Remove oversized PNG delivery from GitHub Pages output.
- Preserve original Figma/source raster exports outside `public`.
- Generate optimized static delivery images locally.
- Remove the Google Fonts render-blocking dependency by self-hosting Figtree.
- Add operator-friendly performance diagnostics and budgets.
- Defer below-fold APR fetching until the protocol section is near the viewport.
- Add missing intrinsic dimensions where this pass touched heavy imagery.

## Implementation

### Source and delivery asset split

Moved heavyweight PNG source exports from `public/assets` into `asset-sources/assets`.

Reason: Vite copies everything in `public` into `dist`. Keeping multi-megabyte source PNGs in `public/assets` meant GitHub Pages shipped them even when the app referenced lighter files.

Source exports remain in Git for future re-encoding. `public/assets` now holds delivery-ready assets.

### Static image pipeline

Added `scripts/build-images.mjs` and the `npm run assets:build` script.

The script:

- Reads original raster exports from `asset-sources/assets`.
- Generates WebP delivery files into `public/assets`.
- Resizes only the oversized visuals where the rendered design does not need the original pixel dimensions.
- Skips unchanged outputs during normal local builds.
- Supports forced regeneration with `FORCE_ASSET_BUILD=1 npm run assets:build`.

The production build now runs:

```bash
npm run clean:metadata && npm run assets:build && tsc -b && vite build
```

### Metadata cleanup

Added `scripts/clean-local-metadata.mjs` and `npm run clean:metadata`.

Reason: `public/assets/.DS_Store` was copied into local `dist/assets`. This is small, but it is bad build hygiene and easy to prevent.

### Performance budgets

Added `scripts/check-performance-budget.mjs` and `npm run perf:budget`.

Current enforced budgets:

- Total `dist`: 12 MiB.
- Largest built file: 900 KiB.
- Built JS gzip total: 130 KiB.
- Built CSS gzip total: 45 KiB.
- No `.DS_Store` files in `dist`.

The full local handoff gate is now:

```bash
npm run check
```

### Lighthouse runner

Added `scripts/perf-audit.mjs` and `npm run perf:audit`.

The script builds the site, starts Vite preview, audits all production pages with Lighthouse, and writes JSON reports to `.performance-reports`, which is ignored by Git.

### Font delivery

Removed the Google Fonts `@import` from `src/styles/global.css`.

Added local Figtree WOFF2 files under `public/fonts` and inline `@font-face` declarations in each HTML entry. The primary latin subset is preloaded from each page head.

Reason: the visual typeface stays the same, but the browser no longer waits on `fonts.googleapis.com` before rendering text.

### Runtime request deferral

Changed the homepage APR request so it starts only when the protocol showcase is near the viewport.

Reason: APR is useful in the protocol section, but it should not compete with first paint, hero rendering, or initial route hydration.

### Image markup

Updated production image references for optimized WebP delivery assets.

Added intrinsic dimensions and a high-priority preload for the homepage hero orb.
