# Corrections

Terra-classic.money should be accurate, neutral, and source-aware.

If you find an error, please open an issue or pull request.

## What To Report

Useful corrections include:

- broken links
- outdated project URLs
- wrong market pairs
- incorrect project status
- missing risk or source context
- typo or clarity fixes
- misleading wording
- duplicate or misplaced listings
- roadmap entries that are stale, unsourced, or too certain

## What To Include

For a useful correction, include:

- the page or file where the issue appears
- the current incorrect text or link
- the proposed correction
- a source link when the correction is factual
- any risk, uncertainty, or conflict of interest maintainer should know

## Pull Request Corrections

If you know the exact fix, submit a pull request.

Common correction files:

| Correction type | File |
| --- | --- |
| Ecosystem listing | `src/data/ecosystem.ts` |
| Verification or infrastructure link | `src/data/ecosystemVerification.ts` |
| Market listing | `src/data/markets.ts` |
| Roadmap item | `src/data/roadmap.ts` |
| Open work package | `src/data/openWork.ts` |
| Shared site link | `src/data/links.ts` |

Keep correction pull requests small and focused.

## Neutrality

Corrections should improve public accuracy, not create hidden promotion.

Maintainers may rewrite submitted copy to keep the site neutral, concise, and official-compatible.
