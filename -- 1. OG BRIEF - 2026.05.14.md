Project Brief: Terra Classic Website / Terra-classic.money 2026

1. Project summary

Terra-classic.money 2026 is an independent, community-maintained, open-source website for Terra Classic.

It is designed as an unofficial public-good website that can serve users, builders, delegators, contributors, investors, and external observers with clear, neutral, and structured information about Terra Classic.

The website should be hosted publicly, preferably through GitHub Pages, with its source code and content managed through GitHub. Community members should be able to propose changes, improvements, corrections, translations, and new entries through issues and pull requests.

The site should not be controlled by validators. Validators may contribute like any other community member, but the editorial and maintenance model should be independent, transparent, and contributor-based.

The long-term ambition is simple:

Build the best independent Terra Classic website first.
If the community and validators consider it useful, neutral, and trustworthy, it can later be proposed again as the official Terra Classic website through governance.

This website is also a continuation of Terra-classic.money v1, which was one of the first serious Terra Classic public websites after the May 2022 Terra collapse and became an important independent wayfinding surface for the ecosystem. Your own playbook describes terra-classic.money as an independent, neutral hub structured for users, builders, validators, and governance, with later governance recognition as a key third-party reference.  ￼

⸻

2. Core idea

Terra Classic is decentralized, but decentralization does not remove the need for a trusted public information layer.

The current Terra Classic ecosystem has many useful tools, dashboards, validators, explorers, dApps, L2 projects, and community groups. But for a newcomer, builder, partner, media person, or investor, the ecosystem is still difficult to understand. There is no single neutral, beautifully structured, source-aware, public website that explains what Terra Classic is today, what exists, what is active, what is experimental, what is community-created, and what risks users should understand.

The State of the Chain report identifies this as a trust-surface problem: Terra Classic can remain technically functional while still being reputationally fragile because onboarding, authority routing, and official-looking surfaces are fragmented.  ￼ It also states that if canonical routing assets such as domains, docs, endpoints, and wallet surfaces are not transparent, the chain inherits routing-integrity and key-person risks.  ￼

Terra-classic.money should directly solve this.

It should become:

An independent, transparent, community-maintained knowledge and wayfinding website for Terra Classic.

It should not be a validator-controlled website.
It should not be a private marketing website.
It should not be a paid-ranking portal.
It should not pretend to be official before governance recognizes it.

Instead, it should be:

* open-source,
* transparent,
* objective,
* community-editable,
* contributor-driven,
* source-aware,
* neutral toward validators and projects,
* clear about what is official, community-made, paid, experimental, or unverified.

The model should be closer to Bitcoin.org than to a company website. Bitcoin.org publicly states that it is an independent open-source project organized through a public GitHub pull-request process, and it explicitly says it is not Bitcoin’s official website because nobody owns the Bitcoin network.  ￼ That is the correct philosophical reference point for Terra-classic.money.

⸻

3. Project mission

The mission of Terra-classic.money is:

To provide Terra Classic with a neutral, transparent, community-maintained public website that helps users understand the ecosystem, discover trustworthy resources, and contribute to the chain without relying on validator-controlled information surfaces.

Secondary mission:

To create a website that can become official-compatible if Terra Classic governance later decides to adopt it.

This means the site must be built from day one with official-readiness in mind:

* public repository,
* visible contribution process,
* transparent maintenance,
* clear correction workflow,
* no hidden sponsorship,
* no paid inclusion in core ecosystem sections,
* no validator favoritism,
* no private control over public narrative,
* clear distinction between neutral ecosystem content and paid commercial listings.

Terra-classic.money is:

* an independent Terra Classic website,

5. What this website is not

Terra-classic.money is not:

* an official Terra Classic website at launch,
* a validator-owned website,
* a governance authority,
* a financial-advice website,
* a token promotion website,
* a paid ranking platform,
* a launchpad,
* an exchange,
* a custody product,
* a wallet,
* a place where paid projects can buy legitimacy inside the main ecosystem section.

This distinction is critical.

If monetization contaminates the main site, the website will lose any chance of being treated as neutral or official-compatible.

6. Governance and contribution model

6.1 GitHub-based community contribution

The website should be managed through GitHub.

Community members should be able to:

* open issues,
* report errors,
* suggest improvements,
* propose new pages,
* submit pull requests,
* suggest new ecosystem entries,
* suggest corrections,
* contribute translations,
* improve documentation,
* propose new educational content.

The contribution model should be visible on the website, especially on the About Terra-classic.money page.

A simple public structure should exist:

* README.md
* CONTRIBUTING.md
* LISTING-POLICY.md
* DONATIONS.md
* L2-LISTING-POLICY.md
* ROADMAP-LISTING-POLICY.md
* SPONSORSHIP-DISCLOSURE.md
* CORRECTIONS.md

Bitcoin.org is a useful reference here: its About page says regular activity is organized through the public pull-request process and managed by co-maintainers, while the site remains an independent open-source project.  ￼

6.2 Independent maintainers, not validator control

The website should be managed by independent community maintainers.

Validators can submit issues, pull requests, corrections, and suggestions like everyone else, but they should not have special editorial control only because they are validators.

This matters because Terra Classic governance is already validator-weighted. If the website is also validator-controlled, it becomes another extension of the same control plane.

The website should instead demonstrate that Terra Classic can have:

* independent public infrastructure,
* transparent contribution rules,
* community-based maintenance,
* editorial neutrality,
* public change history.

6.3 Official-compatible path

At launch, Terra-classic.money should clearly state:

Terra-classic.money is an independent community website. It is not currently the official Terra Classic website. If the Terra Classic community and governance later choose to adopt it, the website can be transitioned into an official-compatible governance model.

If governance adoption ever happens, the site should be ready for:

* public governance proposal,
* defined maintainer rules,
* domain custody / continuity discussion,
* repository access rules,
* transparent editorial process,
* separation between neutral public-good content and paid commercial surfaces,
* public changelog,
* contributor list,
* donations / revenue disclosure.

7. Core neutrality principles

The website must be built around seven principles.

1. Transparency

All content changes should be traceable through GitHub.

Major changes should have visible pull requests or changelog entries.

2. Objectivity

The site should avoid hidden favoritism toward any validator, project, token, meme coin, L2, business, or contributor.

3. No paid inclusion in the core ecosystem section

The main Ecosystem section must remain criteria-based, not payment-based.

This is non-negotiable if the site is intended to remain official-compatible.

4. Clear paid-content separation

Paid listings belong only in clearly labeled paid areas, especially:

* l2.terra-classic.money
* paid L2 project pages
* paid L2 roadmap showcase cards

Paid listings must never appear as neutral endorsements.

5. Decentralized contribution

Anyone can contribute through GitHub.

No single validator group should control what Terra Classic looks like to the outside world.

6. User protection

The website should clearly warn users when something is experimental, paid, unaudited, risky, or community-submitted.

7. No financial advice

The site may educate and list resources, but it must not recommend buying, selling, staking, investing in, or using any token, validator, L2 project, or product as financial advice.

⸻

8. Monetization strategy

The website should be monetized carefully. The monetization model must support maintenance and future development without destroying neutrality.

There are two approved monetization channels:

1. Donations
2. Paid L2 project listings on l2.terra-classic.money

These two channels must be separated from the main neutral ecosystem content.

⸻

9. Monetization channel A: Donations

9.1 Donation model

The website should accept voluntary donations from users, community members, builders, validators, and supporters.

Donation information should be placed on the About Terra-classic.money page, similar in spirit to Bitcoin.org’s donation model. Bitcoin.org states that it is community funded, donations are appreciated, and donations are used to improve the website.  ￼

Suggested wording:

Support Terra-classic.money
Terra-classic.money is an independent community-maintained website. Donations help cover domain costs, development, content updates, research, translations, maintenance, and future improvements. Donations do not buy influence, rankings, listings in the main ecosystem section, or editorial control.

9.2 Donation principles

Donations should be:

* voluntary,
* transparent,
* non-binding,
* non-refundable unless explicitly stated otherwise,
* not connected to editorial decisions,
* not connected to listing approval,
* not connected to validator ranking,
* not connected to governance influence.

A donor should not receive special treatment in the main ecosystem content.

9.3 Suggested donation assets

The donation section may include:

* LUNC address,
* USTC address,
* BTC address,
* USDC / USDT address if desired,
* optional fiat payment method if legal/accounting is clean,
* QR codes,
* short explanation of how donations are used,
* link to public donation / supporter log if maintained.

9.4 Public donation transparency

To preserve trust, the website should publish a simple transparency statement.

Possible options:

Lightweight version

Publish:

* donation addresses,
* intended use of funds,
* no-influence statement,
* maintainer responsibility statement.

Stronger version

Publish quarterly:

* received donations,
* expenses,
* maintenance work completed,
* planned improvements.

Best version

Publish a public “Website Support Log”:

* date,
* asset,
* amount,
* public donor name if donor chooses,
* use category,
* notes.

Do not overcomplicate this at launch. But at minimum, the donation policy must be public.

9.5 Donation disclaimer

Suggested copy:

Donations support the maintenance and development of Terra-classic.money. Donations do not create ownership rights, governance rights, listing rights, promotional rights, editorial rights, or any expectation of financial return.

10. Monetization channel B: l2.terra-classic.money

10.1 Purpose of the L2 subdomain

l2.terra-classic.money should be a separate paid discovery section for Terra Classic-related L2 projects, coins, tokens, meme coins, experimental apps, and community projects that are not eligible for inclusion in the main Ecosystem section.

This distinction is extremely important.

The main site should remain neutral and official-compatible.

The L2 subdomain can be commercial, but it must be visibly separated.

Recommended framing:

L2 Terra Classic Directory
A paid discovery directory for Terra Classic-related L2 projects, tokens, meme coins, experimental apps, and community initiatives. Listings are paid promotional entries and do not represent endorsement by Terra-classic.money, Terra Classic governance, validators, or maintainers.

10.2 Why the L2 subdomain exists

Terra Classic will likely attract many L2 projects, tokens, meme coins, community coins, experiments, and app-layer ideas.

Some may be legitimate.

Some may be early.

Some may be speculative.

Some may never deserve to be in the main Ecosystem section.

But there is still demand for visibility.

l2.terra-classic.money solves this by creating a separate commercial discovery layer:

* projects can pay for visibility,
* users can discover L2 projects in one place,
* the main ecosystem page remains clean,
* official-compatible neutrality is preserved,
* the website has a revenue path.

10.3 Core rule

Payment can buy a listing on l2.terra-classic.money.
Payment cannot buy inclusion in the main Terra Classic Ecosystem section.
Payment cannot buy an endorsement.
Payment cannot buy official status.

This rule should be written publicly.

10.4 What projects can be listed

Eligible paid listings may include:

* Terra Classic L2 projects,
* tokens connected to Terra Classic,
* meme coins connected to Terra Classic,
* experimental Terra Classic apps,
* community-created token projects,
* LUNC / USTC-related app-layer products,
* Juris-related ecosystem projects if appropriate,
* bridges / app-layer integrations if clearly described,
* projects that use Terra Classic branding or assets responsibly.

10.5 Minimum listing requirements

Even paid listings should have minimum standards. Otherwise the directory becomes a scam board.

A project should provide:

* project name,
* logo,
* short description,
* official website,
* official social links,
* project category,
* token ticker if applicable,
* chain / L2 / platform,
* contract address if applicable,
* launch status,
* team / operator disclosure if available,
* risk disclosure,
* roadmap summary,
* contact method,
* confirmation that the listing copy is accurate,
* confirmation that the project is not claiming official Terra Classic status unless governance has granted it.

10.6 Rejection criteria

A paid listing should be rejected if the project:

* impersonates Terra Classic, Terra, validators, or known projects,
* claims official status without proof,
* uses misleading “guaranteed profit” or “risk-free” language,
* hides basic identity/contact information,
* has obviously malicious links,
* promotes phishing, malware, fake wallet connections, or fake airdrops,
* uses deceptive contract/address information,
* refuses to provide basic listing data,
* asks the site to hide that the listing is paid,
* appears to be designed primarily to scam users.

This protects the site.

You cannot prevent every bad project from existing, but you can prevent Terra-classic.money from becoming a paid scam amplifier.

10.7 Listing does not mean endorsement

Every L2 listing page should include a visible notice:

This is a paid listing. Terra-classic.money does not endorse, audit, verify, recommend, or guarantee this project. Users should do their own research before interacting with any token, contract, wallet, website, or community listed here.

10.8 Listing categories

Possible categories:

* L2 protocol
* meme coin
* utility token
* DeFi
* gaming
* NFT / collectibles
* community project
* infrastructure
* DAO / governance
* experimental
* unknown / early-stage

Potential payment methods:

Keep accounting clean. Listing fees are business / website revenue, not user deposits and not funds managed on behalf of third parties.

10.12 Paid listing transparency

Each paid listing should be labeled:

* Paid Listing
* Sponsored Listing
* Featured Paid Listing
* Roadmap Sponsored Entry

Do not hide paid status.

If the website is later proposed as official, this separation will be the difference between “acceptable monetization” and “rejected for pay-to-play.”

11. Main Ecosystem page vs L2 paid directory

This is the most important structural decision in the brief.

Main Ecosystem page

The main Ecosystem page should be non-paid.

It should list:

* core Terra Classic resources,
* wallets,
* explorers,
* dashboards,
* governance resources,
* documentation,
* infrastructure,
* serious ecosystem tools,
* established apps,
* important public-good resources.

Inclusion should be based on objective criteria:

* relevance,
* usefulness,
* safety,
* activity,
* credibility,
* public availability,
* non-duplicative value,
* fit with Terra Classic.

No project should be able to pay its way into the main Ecosystem page.

L2 subdomain

The L2 section is paid and separate.

It should list:

* L2 projects,
* community tokens,
* meme coins,
* early-stage apps,
* speculative experiments,
* projects not appropriate for core ecosystem listing.

This creates a clean separation:

Ecosystem = neutral core directory.
L2 subdomain = paid discovery directory.

That is the right model.

12. Roadmap page monetization

You want paid L2 projects to also be showcased on the Roadmap subpage.

This can work, but only if structured carefully.

The danger

If paid L2 projects appear on a page called “Roadmap,” users may interpret them as part of the official Terra Classic roadmap.

That would damage neutrality.

The solution

The Roadmap page should have two clearly separated areas.

Section 1: Terra Classic Public Roadmap

This section should not be paid.

It should include:

* protocol upgrades,
* governance-recognized initiatives,
* public infrastructure work,
* major ecosystem milestones,
* official or official-compatible items,
* source-linked roadmap items.

Section 2: L2 & Community Project Roadmap

This section can contain paid entries.

Label it clearly:

L2 & Community Project Roadmap
This section includes paid project-submitted roadmap entries from Terra Classic-related L2 projects, tokens, meme coins, and community initiatives. These entries are not official Terra Classic roadmap items unless separately confirmed by governance.

Each paid roadmap card should include:

* project name,
* milestone,
* target date,
* status,
* project category,
* paid listing label,
* link to L2 profile,
* disclaimer.

Suggested labels:

* Planned
* In progress
* Live
* Delayed
* Completed
* Unverified
* Project-submitted
* Paid listing

Mandatory disclaimer

Project-submitted roadmap entries are paid promotional submissions. Terra-classic.money does not verify future delivery, guarantee milestones, or endorse the listed projects.

This protects the main website.

4. How the community can contribute.

6. How paid listings work.

About Terra-classic.money

What is Terra-classic.money?

Terra-classic.money is an independent, community-maintained website for Terra Classic. It exists to help users, builders, delegators, and external observers understand Terra Classic through clear information, ecosystem wayfinding, and transparent public resources.

Is this the official Terra Classic website?

No. Terra-classic.money is not currently the official Terra Classic website. It is an independent community project. If the community and validators later decide that it should become official, this can be proposed through Terra Classic governance.

Who maintains Terra-classic.money?

The website is maintained by independent community contributors. The website is not controlled by validators. Contributions are managed publicly through GitHub.

How can I contribute?

Anyone can open an issue or pull request on GitHub to suggest corrections, improvements, new content, translations, or new resources.

How is the website funded?

The website is funded through voluntary donations and paid listings on l2.terra-classic.money.

Do donations buy influence?

No. Donations do not buy influence, editorial control, ecosystem placement, rankings, or governance status.

What is l2.terra-classic.money?

l2.terra-classic.money is a separate paid listing directory for Terra Classic-related L2 projects, tokens, meme coins, and experimental community projects that are not part of the main neutral Ecosystem directory.

Are paid L2 listings endorsements?

No. Paid listings are promotional entries. They are not endorsements, audits, recommendations, or guarantees.

How are corrections handled?

Errors can be reported through GitHub or the website correction form. Corrections are reviewed publicly where possible.

The monetization model must be brutally clean.

Acceptable monetization

* donations,
* paid L2 listings,
* paid L2 roadmap entries,
* clearly labeled featured L2 placements,
* optional sponsor/supporter section,

Not acceptable

* paid inclusion in main Ecosystem section,
* paid “official” labels,
* paid validator favoritism,
* paid ranking manipulation,
* hidden sponsored links,
* paid placement disguised as neutral editorial,
* paid roadmap entries shown as core Terra Classic roadmap,
* token promotion without disclosure,
* listing projects that obviously impersonate Terra Classic.

The website’s future official potential depends on this separation.

20. Final brief statement

Terra-classic.money 2026 is an independent, open-source, GitHub-managed Terra Classic website designed to become the ecosystem’s most transparent, neutral, and useful public information layer.

It continues the work of Terra-classic.money v1, the first serious post-crash Terra Classic website, but rebuilds it with a stronger governance model: public contribution, independent maintainers, no validator control, transparent corrections, and official-compatible structure.

The website will be monetized through two carefully separated channels:

1. Voluntary donations, shown on the About page, used to support maintenance and improvement of the website.
2. Paid listings on l2.terra-classic.money, a separate commercial discovery area for Terra Classic-related L2 projects, coins, tokens, meme coins, and experimental community initiatives that are not part of the main neutral Ecosystem directory.

The main website must remain neutral.
The core Ecosystem page must not be pay-to-play.
Paid L2 listings must be clearly labeled.
Roadmap paid entries must be separated from official or core Terra Classic roadmap content.
Nothing paid should imply endorsement, governance recognition, official status, safety, or investment merit.

This structure gives Terra Classic a better public website while giving the project a clean, defensible monetization path.
